## 1. 一句话定位

> **"你只管散步和说话，AI 替你看、替你记，回家送你一份 360° 的散步纪念品。"**

步语是一款**音频优先的 AI 散步同伴 App**。用户戴影石相机 + 蓝牙耳机出门散步，手机放口袋。AI 通过相机视角看世界、通过耳机与用户对话；散步结束后，AI 自动生成一段融合 360° 视频片段、地图轨迹、AI 海报与对话精华的**短视频纪念品**。

## 2. 核心产品决策（已确认）

| # | 决策 | 取代/收敛了什么 |
|---|---|---|
| D1 | **散步过程中，用户注意力给世界，不给手机**（Q1=A）| 推翻 brief 中"上半屏地图=主舞台"的实时舞台叙事 |
| D2 | **核心交付物 = 散步结束后的纪念品**，不是过程中的实时地图共创 | 地图共创从"实时表演"降级为"异步 artifact" |
| D3 | **AI 主动开口 ≤ 3 次 / 10 分钟**，且**主动开口 = 地点推荐**（Q5=C）| 取消"AI 主动指出哇时刻 / 提示走神 / 主动画图标"等行为 |
| D4 | **氛围 / 灵感捕捉走"用户被动触发"路径**（用户说"记一下"才记） | 取消 AI 自主判断"这个时刻值得记" |
| D5 | **推荐可协商**：用户对地点推荐不满意可继续 chat → 换一个 | brief 中没有的能力，必做 |
| D6 | **不做 SVG 图标库 + 语义匹配，纪念品用 diffusion 生图一次性出**（Q8=A）| 取消 brief 中 200–500 个手绘 SVG 资产、AI 摆放、动效等子工程 |
| D7 | **AI 架构 = ReAct Agent + OpenAI 兼容 API + 工具集**，模型走 `http://100.99.139.20:18141`（端口内任意模型可选）| 取代 brief 中 VLM/LLM/语义匹配的多模块组合 |
| D8 | **App 平台 = iOS 原生**（Q9=iOS） | 不做 Android、Flutter、RN |
| D9 | **纪念品形态 = 自动剪辑短视频，长图海报作为兜底**（Q10=E）| 不做独立 H5 |

## 3. 用户故事（MVP 主场景）

**周末下午，玄武湖边 30 分钟散步。**

1. 用户出门前打开 App，连接影石相机（WiFi P2P）+ 蓝牙耳机，按下"开始散步"。手机进口袋。
2. 散步开始。GPS 持续记录轨迹；相机持续录制视频并向手机推送预览流；麦克风待机听语音指令。
3. **被动应答**：用户说"嘿，那是什么花？" → AI 回溯最近若干秒相机帧 → VLM 分析 → 耳机回答："看起来像关山樱，江户时代的园艺品种。"
4. **主动推荐（A 类，≤3 次/10 分钟）**：当用户接近一个有故事的 POI（高德 around_search 命中），AI 在耳机里说："前面 200 米有家民国老茶馆，要不要绕过去看看？"用户可以说"换一个"或"再聊聊"。
5. **被动捕捉（B 类）**：用户说"记一下我刚才说的那个想法" → Agent 调用 `record_moment(type="idea", context=最近 N 秒对话与位置)` → 静默记录，无打扰。
6. **方向指引**：用户说"带我去湖那边" → AI 在耳机里说："直线方向往东南，大约 600 米。"（不强制路线）
7. 30 分钟后用户说"散步结束"。App 后台开始生成纪念品。
8. **纪念品**：30–60 秒短视频，包含自动精选的 360° 视频片段、地图轨迹动画、对话精华字幕、AI 生成的"散步海报"作为定格收尾。可一键分享。

## 4. AI 行为契约

### 4.1 硬约束（写进 system prompt）

- **主动开口频率上限**：≤ 3 次 / 10 分钟（被动应答不计入）
- **主动开口的合法触发**：仅"附近有值得推荐的地点"一种。其他类型主动一律禁止。
- **沉默是默认状态**。没事不要说话。
- **每次推荐被拒绝后**：可以 chat about it，可以换一个，但**该次主动配额已消耗**。
- **被动捕捉的触发**：必须由用户语音明确指令（"记一下" / "这个想法挺有意思" / "标个点"等明显语义信号）。

### 4.2 ReAct Agent 工具集

| 工具 | 用途 |
|---|---|
| `amap_around_search(lat, lng, type, radius)` | 周边 POI 搜索（推荐场景）|
| `amap_text_search(query, region)` | 关键词地点搜索（用户问"附近有咖啡馆吗"）|
| `amap_direction_walking(from, to)` | 步行方向（用作"指引"，仅取方向与距离，不取详细路线）|
| `amap_geo / amap_regeocode` | 坐标-地址互转 |
| `get_camera_frame(timestamp_offset)` | 从滑动窗口取相机帧（默认最近一帧；可回溯）|
| `analyze_frame_vlm(frame, question)` | VLM 分析图像 |
| `record_moment(type, context, location)` | 记录关键时刻（B 类捕捉）|
| `speak_to_user(text)` | TTS 在耳机播报 |

### 4.3 时空对齐机制

- 相机帧、GPS 点、对话事件全部带 UTC 时间戳
- `record_moment` 记录的 location 是**事件发生当时的 GPS**，不是当前位置
- 滑动窗口：保留最近 5 分钟的相机帧（采样率约 1–2 fps，纪念品所用完整视频另由相机本机录制）+ 最近 30 分钟的 GPS / 对话历史

### 5.2 关键数据流

**散步进行中（实时循环）**：

1. `CameraBridge` 持续接收相机预览流，按 1–2 fps 采样存入滑动窗口；同时触发相机本机录制。
2. `LocationSvc` 持续记录 GPS 点。
3. `AudioIO` 监听用户语音，STT 转文本送入 `AgentRuntime`。
4. `AgentRuntime` 在两种 trigger 下激活：
   - **用户语音输入**（被动应答 / 被动捕捉）
   - **位置变化定时检查**（每 ~30 秒检查一次"是否值得发起一次主动推荐"）
5. Agent 通过 ReAct 推理调用工具，最终决定是 `speak_to_user` / `record_moment` / 静默。

**散步结束（后台一次性流程）**：

1. `WalkSession` 切到 `ending` 状态，停止相机录制，从相机下载完整视频文件。
2. `KeepsakeBuilder.collectMaterials()` 汇总 GPS 轨迹、moments、对话历史、视频文件路径。
3. `LLMClient.generateScript(materials)` 一次调用，返回结构化"剧本"：
   ```json
   {
     "title": "...",
     "narration": "...",
     "poster_prompt": "...",
     "video_clips": [{"start": 120.3, "duration": 4.0, "caption": "..."}],
     "highlight_moments": [...]
   }
   ```
4. **并行渲染**：
   - `DiffusionClient.generate(poster_prompt)` → 海报图
   - `MapRenderer.exportTrack(...)` → 轨迹图
   - `AVFoundation` 按剧本截取视频片段
5. **拼合**（AVFoundation `AVMutableComposition`）：开场地图轨迹动画 → 视频精选 + 字幕 → 海报定格。
6. **降级路径**：任一步失败超过重试上限 → 切到长图海报模式（仅需 diffusion 海报 + 轨迹图 + 文案拼合）。

## 6. 错误处理 / 降级策略

| 失败点 | 降级策略 |
|---|---|
| 相机 WiFi 断连 | 提示用户重连；GPS 与对话继续记录；纪念品生成时缺少视频片段则跳过该模块 |
| LLM API 超时 / 不可用 | 主动推荐静默跳过；被动应答给一句兜底话术；纪念品阶段失败则降级长图 |
| VLM 分析失败 | Agent 兜底回复"我没看清"，不阻塞对话流 |
| 高德 API 失败 | 推荐轮空跳过；轨迹仍能用 GPS 点本地绘制 |
| 视频合成失败 | 降级到长图海报（必须保证至少有一种纪念品产出）|
| 相机预览流与本机录制冲突 | **待验证假设**（见 §9）。若冲突：录制优先，预览流降级到低帧率截图轮询 |

## 7. 测试策略

- **AgentRuntime 单元测试**：mock 工具集，验证 ReAct 在常见输入下的行为符合契约（特别是主动配额、推荐协商循环、被动捕捉触发条件）
- **集成测试（模拟散步）**：脚本回放一段 GPS 轨迹 + 模拟语音输入 + 桩相机帧 → 跑完整 session → 断言纪念品文件生成
- **端到端真机测试**：玄武湖至少 3 次完整散步走查，每次问题立刻迭代
- **降级路径测试**：手动 kill 各依赖（断 WiFi、停 LLM、404 高德）观察是否优雅降级
- **Demo 彩排**：比赛前一周开始每天一次完整 demo 流程，覆盖最佳路径与至少一个降级路径

## 9. 待验证假设（需要影石支持团队 / 用户回头确认）

1. **影石相机是否支持"WiFi 预览流 + 本机录制"并发**？（暗礁 1，最高优先级）
   - 如不支持：架构需要切换到"周期性截图 + 全程录制"或"分时段切换"
2. **影石 SDK 在 iOS 原生（Swift / Objective-C）的官方支持完备度**
3. **LLM API endpoint 在 demo 现场的网络可达性**（100.x 看似 Tailscale / 内网，需确认 demo 现场如何接入）
4. **VLM 模型选型**：endpoint 内可选模型中哪一款的视觉能力最适合户外场景（樱花、雕塑、店招识别）
5. **TTS 实时性**：用 LLM endpoint 自带 TTS 还是 iOS 系统 TTS？前者音质好但延迟可能超过 1 秒
6. **配乐版权**：纪念品短视频用什么背景音乐（无版权库 / iOS 内置 / 用户选）

## 10. 显式不做（避免范围回锅）

- ❌ Android / Flutter / React Native 版本
- ❌ 散步过程中地图上的 SVG 图标叠加 / 长出来动效（手机在口袋里没人看）
- ❌ 轨迹艺术化笔触（基础线即可）
- ❌ 360° 哇时刻主动提示
- ❌ AI 主动判断"这个时刻值得记"（B 类只走用户被动触发）
- ❌ AI 主动提示用户走神 / 走错方向
- ❌ 长按图标编辑 / AI 误识别学习
- ❌ 元素时间衰减、Zoom 分层、思考雾团聚类
- ❌ 跟着别人地图走、多人共创
- ❌ Turn-by-turn 导航
- ❌ 实时 diffusion 生图（仅纪念品阶段使用）
- ❌ 高德 personal-map 二维码方案
- ❌ 心情/对话情感影响轨迹颜色
- ❌ 独立 H5 分享页（只做短视频 + 长图兜底）

## 11. 成功标准

- **可运行**：能在玄武湖完整走完一次 30 分钟散步，全程无崩溃
- **AI 不烦人**：30 分钟散步中，AI 主动开口次数 ≤ 9 次，被动应答有问必答
- **纪念品有冲击力**：评委首次看到纪念品短视频时，"哇"一下
- **降级稳健**：拔网线 / 关相机的降级路径，demo 现场不翻车
- **影石特色明显**：纪念品中 360° 视频镜头语言可被识别（评委看出"这是 360° 拍的"）

## Slide 2 — 问题 ："phone-as-stage tax"

- 散步 App 的隐性税：你被迫盯着手机
- 注意力本来该给世界，被舞台抢走
- AI 的位置应该在**眼睛之外**
- **推荐视觉**：左右对照插画 — 左：低头族走路；右：耳机+相机+口袋手机
- **演讲对应**：0:30–0:45

---

## Slide 3 — 洞察 ：音频优先 + 主动配额

- 主动开口 ≤ 3 次 / 10 分钟
- 主动开口的唯一合法场景 = 推荐附近地点
- 默认状态 = **沉默**
- 被动捕捉 = 用户开口才记
- **推荐视觉**：一条时间轴，30 分钟散步上只有 3 个气泡（"民国茶馆"、"湖东亭"、"老梅园"）；其余皆静默
- **演讲对应**：0:45–1:00

- iOS 原生 + ReAct Agent + 8 工具 + OpenAI 兼容端点
- Insta360 SDK · CoreLocation · 高德 SDK · AVFoundation

- **演讲对应**：3:30–4:00（主路径）/ 1:30–2:00（降级路径）

## Slide 5 — AI 行为契约

- **硬约束 1**：≤ 3 主动 / 10 分钟（写进 system prompt + 独立配额计数器双保险）
- **硬约束 2**：主动 = 仅地点推荐
- **硬约束 3**：被动捕捉只走用户语音明确指令
- **可协商**：推荐被拒 → 可换、可聊，但配额已扣
- **推荐视觉**：四张约束卡片 2×2 排布，红框 = 硬约束，蓝框 = 可协商
- **演讲对应**：架构问答时引用

## Slide 6 — 工具集 + ReAct 循环

- 8 个工具：3 高德 + 2 相机/视觉 + 3 交互
- Agent 自主决定调谁、按什么顺序、何时闭嘴
- **运行时**会拒绝超额的主动 speak 调用（双保险）
- **推荐视觉**：左侧 8 个工具的图标列表 + 右侧 ReAct 思维链小图（Thought → Action → Observation 循环 3 圈）
- **演讲对应**：3:30–4:00

---

## Slide 7 — Insta360 三个角色

- **第一视角眼睛**：给 VLM 提供视觉
- **本机录制**：完整高质量素材，不依赖手机带宽
- **纪念品的灵魂**：360° 镜头语言是辨识度最高的部分
- 拔掉 Insta360 → 产品 = 普通 vlog 工具
- **推荐视觉**：相机居中，三条线连出去到三个图标（眼睛 / 硬盘 / 影片胶卷）
- **演讲对应**：4:00–4:30

## Slide 9 — 降级策略矩阵

- 相机断 → GPS + 对话继续，纪念品缺视频段
- LLM 不通 → 主动静默，被动给兜底
- 视频合成失败 → 长图海报兜底
- **永远保证至少一份纪念品**
- **推荐视觉**：3 列 ×3 行表格（失败点 / 触发条件 / 降级动作），第 4 行红字 "保底：必出一份纪念品"
- **演讲对应**：1:30–2:15（degradation）/ Q&A 时引用

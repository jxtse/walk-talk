# Demo Script — 降级路径 / Degradation path

**时长目标**：2–3 分钟（默认 2:30）
**何时启用**：现场 Wi-Fi / VPN / LLM endpoint 任一不可用、相机连不上、或主路径演到一半翻车
**前置条件**：
- 笔记本本地播放器加载好两段视频：`walk-3-keepsake.mp4`（happy-path 成品）+ `walk-2-keepsake.mp4`（adversarial 成品）
- 二者**都是离线文件**，不需要网络
- 配套幻灯片 `deck-outline.md` 就位

**口播语种**：中文普通话

**演讲者心态备忘**：
- **不解释**为什么主路径不演了。直接进入"我们今天选纪录片模式"。
- **降级 ≠ 失败**。降级路径的存在本身，就是产品成熟度的证据。把这一点讲出来。

## 1:30 – 2:15 ｜ 架构 + 降级讲解（45 秒）

`[SLIDE 4: 架构图]`

> 它跑得起来的原因是这套架构：
>
> **ReAct Agent + 8 个工具 + OpenAI 兼容端点**。
>
> 高德、影石相机、TTS、record_moment 都是工具。Agent 自己决定调谁。

`[SLIDE 9: 降级策略表]`

> 我特别想讲的是**降级策略**——这是我们花了同样多时间打磨的部分。
>
> - 相机断了：GPS 和对话继续，纪念品里少一段视频片段，但**一定有海报**
> - LLM 不通：主动推荐静默，被动应答给固定兜底，**不会卡死**
> - 视频合成失败：**降级到长图海报**——永远保证至少一份纪念品
>
> 这不是嘴上说说。

## 开播前 60 秒诊断流程（决定走主路径 or 降级）

> **演讲者一人秒级判断用。**

1. 手机连相机：相机预览缩略图 5 秒内出来 → ✅ 否则 → 降级
2. Tailscale 状态：绿色 → ✅ 否则 → 降级
3. App 内 "ping LLM" 按钮：返回 200 → ✅ 否则 → 降级
4. 蓝牙耳机：戴上听到一声 prompt → ✅ 否则 → 降级
5. **任一项 ✗ → 立即切降级脚本**。不要试图修，不要在评委面前 debug。
